# API Reference

All REST endpoints exposed by the FlowMind AI backend. Base URL: `http://localhost:8000/api/v1`

Interactive docs (auto-generated by FastAPI): `http://localhost:8000/api/v1/openapi.json`  
Swagger UI: `http://localhost:8000/docs`  
ReDoc: `http://localhost:8000/redoc`

---

## Authentication

The current version has no authentication on the API. All endpoints are publicly accessible. For production use, add an API key or JWT middleware to FastAPI before exposing the backend.

## Workflows

### `POST /api/v1/workflows`

Create a new workflow.

**Request body**

```json
{
  "name": "My Workflow",
  "description": "Optional description",
  "graph_data": {
    "nodes": [
      {
        "id": "node_001",
        "type": "inputNode",
        "position": { "x": 60, "y": 200 },
        "data": {
          "label": "Input",
          "fields": [
            { "id": "f1", "name": "customer_id", "type": "text" }
          ]
        }
      },
      {
        "id": "node_002",
        "type": "outputNode",
        "position": { "x": 300, "y": 200 },
        "data": { "label": "Output", "outputType": "api" }
      }
    ],
    "edges": [
      { "id": "e1", "source": "node_001", "target": "node_002" }
    ]
  }
}
```

**Response** `201 Created`

```json
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "name": "My Workflow",
  "description": "Optional description",
  "graph_data": { ... },
  "status": "draft",
  "version": "1.0",
  "created_at": "2025-01-15T10:30:00Z",
  "updated_at": "2025-01-15T10:30:00Z"
}
```

    "description": "...",
    "status": "draft",
    "version": "1.0",
    "created_at": "...",
    "updated_at": "..."
  }

### `GET /api/v1/workflows/{workflow_id}`

Get a specific workflow by ID.

**Path parameters**

| Param | Description |
|---|---|
| `workflow_id` | UUID of the workflow |

**Response** `200 OK` — same shape as create response

**Error** `404 Not Found`

```json
{ "detail": "Workflow not found" }
```

### `PUT /api/v1/workflows/{workflow_id}`

Update a workflow. Only fields included in the request body are updated (partial update).

**Request body** (all fields optional)

```json
{
  "name": "Updated Name",
  "description": "Updated description",
  "graph_data": { ... },
  "status": "active"
}
```

**`status` values:** `draft`, `active`, `paused`, `archived`

**Response** `200 OK` — updated workflow object

---

### `DELETE /api/v1/workflows/{workflow_id}`

Delete a workflow and all its associated executions.

**Response** `204 No Content`

---

### `POST /api/v1/workflows/{workflow_id}/execute`

Execute a workflow from the UI (ExecutionModal). Accepts structured per-node input.

**Request body**

```json
{
  "input_data": {
    "node_inputs": {
      "node_001": {
        "customer_id": { "content": "cust_123" },
        "identity_document": {
          "content": "data:application/pdf;base64,JVBERi0x...",
          "filename": "passport.pdf"
        }
      }
    }
  }
}
```

The `node_inputs` key maps node IDs to their field values. Each field value is an object with `content` (the actual value or base64 data URI) and optionally `filename` (for file uploads).

**Response** `200 OK`

```json
{
  "id": "exec-uuid",
  "workflow_id": "workflow-uuid",
  "status": "completed",
  "input_data": { ... },
  "output_data": { "decision": "approved", "confidence": 0.91 },
  "execution_trace": [
    {
      "node_id": "node_001",
      "node_type": "inputNode",
      "result": { "inputs": { "customer_id": "cust_123" }, "timestamp": "..." },
      "timestamp": "2025-01-15T10:30:00Z"
    },
    {
      "node_id": "node_002",
      "node_type": "decisionNode",
      "result": {
        "decision": "approved",
        "confidence": 0.91,
        "threshold": 0.8,
        "reasoning": "...",
        "key_findings": [...]
      },
      "timestamp": "2025-01-15T10:30:02Z"
    }
  ],
  "error_message": null,
  "started_at": "2025-01-15T10:30:00Z",
  "completed_at": "2025-01-15T10:30:03Z"
}
```

**On failure:**

```json
{
  "status": "failed",
  "error_message": "Node 'Field Validation' failed: Validation failed: 'email' is required but missing",
  "execution_trace": [ ... ],
  "output_data": null
}
```

### `POST /api/v1/workflows/{workflow_id}/trigger`

Trigger a workflow from an external system. Accepts any flat JSON body. The Input node maps top-level keys to its defined fields by name.

**Request body** — any JSON object

```json
{
  "customer_id": "cust_123",
  "amount": 4999.00,
  "transaction_type": "purchase"
}
```

**Response** `200 OK`

```json
{
  "workflow_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "execution_id": "exec-uuid",
  "status": "completed",
  "output": {
    "decision": "approved",
    "confidence": 0.91,
    "reasoning": "Transaction amount is within normal range..."
  }
}
```

**Status values in response:**

| Status | Meaning |
|---|---|
| `completed` | Workflow ran to completion |
| `failed` | A node returned an error; execution halted |
| `completed` (with review data) | Paused at a Human Review node |

**cURL example:**

```bash
curl -X POST http://localhost:8000/api/v1/workflows/3fa85f64-5717-4562-b3fc-2c963f66afa6/trigger \
  -H "Content-Type: application/json" \
  -d '{
    "customer_id": "cust_123",
    "full_name": "Sarah Connor",
    "annual_income": 85000,
    "credit_score": 742
  }'
```

**Python example:**

```python
import httpx

response = httpx.post(
    "http://localhost:8000/api/v1/workflows/WORKFLOW_ID/trigger",
    json={
        "customer_id": "cust_123",
        "annual_income": 85000,
        "credit_score": 742,
    }
)
result = response.json()
print(result["status"])   # "completed"
print(result["output"])   # final node output
```

## Graph Data Schema

The `graph_data` object stored in a workflow follows this structure:

```typescript
interface GraphData {
  nodes: WorkflowNode[];
  edges: WorkflowEdge[];
}

interface WorkflowNode {
  id: string;           // unique node ID within the workflow
  type: NodeType;       // one of 14 node type strings
  position: { x: number; y: number };
  data: NodeData;       // node-specific config (see Node Reference docs)
}

interface WorkflowEdge {
  id: string;
  source: string;       // source node ID
  target: string;       // target node ID
  sourceHandle?: string; // required for routing nodes: "approved", "rejected", "pass", "fail", "case_0", etc.
  targetHandle?: string; // optional
}
```

### `sourceHandle` Values by Node Type

| Node | sourceHandle values |
|---|---|
| Decision | `approved`, `rejected` |
| Rule Engine | `pass`, `fail` |
| Switch Router | `case_0`, `case_1`, ..., `default` |
| All other nodes | (omit — or any value, it's ignored) |

## Input Node Field Types

When using `/trigger`, the Input node maps top-level JSON keys to fields by `name`. When using `/execute` (UI), each field is nested under `node_inputs.{nodeId}.{fieldName}`:

| Field type | `/trigger` body value | `/execute` content value |
|---|---|---|
| `text` | `"hello"` | `"hello"` |
| `number` | `4999.00` | `"4999.00"` (as string) |
| `json` | `{ "key": "value" }` | `'{"key": "value"}'` (as JSON string) |
| `file` | Base64 data URI | Base64 data URI |

## Error Responses

All errors follow FastAPI's standard format:

```json
{ "detail": "Human-readable error message" }
```

| HTTP Status | When |
|---|---|
| `400 Bad Request` | Invalid request body (Pydantic validation failure) |
| `404 Not Found` | Workflow or execution ID does not exist |
| `422 Unprocessable Entity` | Request schema doesn't match expected shape |
| `500 Internal Server Error` | Unexpected server error |

## Webhook Integration Example

To integrate FlowMind AI with an existing system as a webhook receiver:

1. Save your workflow in the UI and note the workflow ID from the URL or the Load panel
2. Call `/trigger` whenever your event fires:

```javascript
// Node.js example
async function processNewApplication(applicationData) {
  const response = await fetch(
    `http://your-server:8000/api/v1/workflows/${WORKFLOW_ID}/trigger`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(applicationData),
    }
  );
  const result = await response.json();
  
  if (result.status === 'completed') {
    const output = result.output;
    // Handle completed workflow output
  } else if (result.status === 'failed') {
    // Handle failure
    console.error('Workflow failed:', result.output);
  }
}
```

## Rate Limits and Timeouts

There are currently no rate limits. The practical limit is the Gemini API's quota on your key and the HTTP request timeout configured per HTTP node (default 30s).

For production deployments, add rate limiting at the nginx/load balancer level rather than in the application.
