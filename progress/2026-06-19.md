## Feature 3 ŌĆö User Config Page, Platform Model Settings & Account Activation

### Account Activation Flow
- New users sign up in **inactive state** (`is_active = false` default)
- Inactive non-admin users see a **"Pending Approval"** screen with next-steps info
- Admin sees a **Pending Approval** section (amber) in the Users tab listing all accounts awaiting approval
- Admin clicks **Approve** to activate ŌĆö maps to `PATCH /admin/users/{id}` with `{ is_active: true }`
- Attempting to execute/trigger a workflow while inactive returns HTTP 403 with a friendly message

### Platform Settings (Admin)
Admin can configure platform-wide default API keys and models via the Admin Panel ŌåÆ **Platform Settings** section.

**New DB model:** `PlatformSetting` (single-row, id=1)
- Fields: `openai_api_key`, `openai_model`, `anthropic_api_key`, `anthropic_model`, `gemini_api_key`, `gemini_model`

**New endpoints:**
- `GET /admin/platform-settings`
- `PUT /admin/platform-settings`

### User Config Page
Every user can set their own API keys and model selections via the **Settings** button in the toolbar.

**New endpoint:** `GET /config/me` and `PUT /config/me`

**Key resolution chain (per-execution):**
```
User's own key ŌåÆ Platform default (admin) ŌåÆ Env var ŌåÆ Hardcoded fallback
```

**UI:** Three provider cards (OpenAI, Anthropic, Gemini) with:
- Toggle: Use platform default / Use custom key
- Masked API key input with show/hide
- Model dropdown (per-provider options)
- Shows effective model and key source

**Executor changes:**
- `execute_workflow(graph_data, input_data, resolved_keys=None)` ŌĆö resolved keys passed in from the API layer
- Keys stored in `context["_resolved_keys"]` and picked up by `_get_llm(context=context)` at node execution time
- Fully thread-safe (no shared instance state)

### Toolbar additions
- **Settings** button ŌåÆ routes to ConfigPage
- **Credits badge** with fill bar (green / amber / red based on remaining)
- **Admin** button (admin role only) ŌåÆ routes to AdminPanel
- User email display + **Logout** button

### Live API Panel
- Displays the user's Bearer API token (masked, with copy button)
- cURL snippet now includes `Authorization: Bearer <token>` header
