## Fix 1 ‚Äî Admin Seed Not Running

**Symptom:** Login with `admin@flowmind.com` returned 401 even after setting `ADMIN_PASSWORD` in `.env`.

**Root cause:** `ADMIN_EMAIL` was missing from `backend/.env`. `_seed_admin()` in `main.py` checks both `ADMIN_EMAIL` and `ADMIN_PASSWORD` and returns early if either is absent, so the admin row was never inserted.

**Fix:** Added `ADMIN_EMAIL="admin@flowmind.com"` to `backend/.env`.

---

## Fix 2 ‚Äî `passlib` / `bcrypt` Version Mismatch

**Symptom:** Backend crashed at startup with `AttributeError: module 'bcrypt' has no attribute '__about__'` when `_seed_admin()` called `hash_password()`.

**Root cause:** `passlib 1.7.4` tries to read `bcrypt.__about__.__version__` at import time. `bcrypt 4.1+` removed `__about__`. The installed bcrypt was newer than passlib expected.

**Fix:** Pinned `bcrypt==4.0.1` in `backend/requirements.txt`. Rebuilt the Docker image with `docker compose up -d --build backend`.

## Feature 3 ‚Äî Token-Based Billing for AI Nodes

AI nodes (AI Agent, Verification, Decision) are now billed based on actual LLM token consumption instead of a flat fee.

### Formula

```
credits = max(1, ceil(total_tokens / 1_000))
```

- 1 credit = 1,000 tokens (input + output combined)
- Minimum 1 credit per call regardless of response size
- Falls back to 1 credit if the provider does not return token metadata

### How token counts are captured

`NodeExecutor._extract_token_usage(response)` reads:
1. `response.usage_metadata` ‚Äî standard LangChain 0.2+ `AIMessage` attribute (`input_tokens`, `output_tokens`, `total_tokens`)
2. `response.response_metadata["usage_metadata"]` ‚Äî Gemini-specific fallback (`prompt_token_count`, `candidates_token_count`, `total_token_count`)

All three AI node methods (`execute_ai_node`, `execute_verification_node`, `execute_decision_node`) now include `_token_usage: {prompt_tokens, completion_tokens, total_tokens}` in their result dict. Token counts are also logged at INFO level.

### Credit charging

`_charge_credits()` now branches:
- **AI nodes** ‚Üí read `_token_usage` from result ‚Üí apply formula ‚Üí store description as `"AI Agent ¬∑ reasoning ¬∑ 1,250 tokens (950 in + 300 out)"`
- **All other nodes** ‚Üí unchanged flat rate

**Files changed:**
- `backend/app/services/workflow_executor.py` ‚Äî `_extract_token_usage()` static method, `_token_usage` in AI node results
- `backend/app/api/workflows.py` ‚Äî `_tokens_to_credits()`, `_TOKEN_BASED_NODES` set, updated `_charge_credits()`

## Feature 4 ‚Äî Free AI Usage When User Supplies Their Own Key

If a user has configured their own API key for a provider (Gemini, OpenAI, Anthropic) in their Settings page, executions that use that key are **not charged credits**.

### How it works

`_resolve_api_keys()` now returns a `_key_sources` dict alongside the keys:

```python
"_key_sources": {
    "gemini":    "user" | "platform" | "env",
    "openai":    "user" | "platform" | "env",
    "anthropic": "user" | "platform" | "env",
}
```

In `_charge_credits()`, before billing any AI node, the source for the active provider is checked. If `source == "user"`, billing is skipped entirely for that node (`continue`) with a log line: `Credits skipped | node=aiNode | user owns gemini key`.

The `resolved_keys` dict (including `_key_sources`) is now passed through from both `/execute` and `/trigger` endpoints to `_charge_credits`.

**Note:** When per-node model selection is added (Phase 3 roadmap), the `provider = "gemini"` mapping inside `_charge_credits` will need to read the node's configured provider instead.

**Files changed:**
- `backend/app/api/workflows.py` ‚Äî `_resolve_api_keys()` source tracking, `_charge_credits()` key-source check

## Credit Billing Summary (After Today)

| Node | Billing model | Cost |
|---|---|---|
| AI Agent | Token-based | 1 cr / 1K tokens (min 1) |
| Verification | Token-based | 1 cr / 1K tokens (min 1) |
| Decision | Token-based | 1 cr / 1K tokens (min 1) |
| Indian KYC | Flat | 15 cr |
| HTTP Request | Flat | 2 cr |
| Code Runner | Flat | 2 cr |
| Transform / Rule / Validator / Switch / Formatter / Aggregator / Human Review | Flat | 1 cr |
| Input / Output | Free | 0 cr |

**Own-key exemption:** All token-based node charges are waived when the user supplies their own provider API key.
